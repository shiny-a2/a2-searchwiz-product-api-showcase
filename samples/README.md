# Sample Code

This directory is reserved for Phase 3 public-safe snippets. The samples will demonstrate snapshot API workflow-control approach without mirroring production source code.

## Planned Samples

- `php/product-snapshot-dto.php` — generic product document shaping for feed/search consumers.
- `php/request-access-policy.php` — simplified request boundary checks without real endpoints or allowlist values.
- `php/snapshot-job-state.php` — small full/delta snapshot rebuild state transitions.

## Sample Rules

- Use fictional product data, identifiers, URLs, categories, and attributes.
- Keep snippets short enough for review.
- Explain what each snippet demonstrates.
- Exclude real endpoints, IP addresses, provider details, product IDs, SKUs, GTIN values, image URLs, prices, stock data, table names, option names, hook names, nonce names, logs, exports, and credentials.
- Do not include production response contracts or upload-ready catalog payloads.

## Status

Phase 3 is planned. These samples will be intentionally simplified and should be read as review samples, not as production API code.
